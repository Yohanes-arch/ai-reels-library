# AI Reels Library

Local-first React/Vite app for turning saved Instagram AI reels and DM-shared links into a searchable knowledge library.

The current MVP is focused on finding high-signal MCP, TradingView, GitHub, coding-agent, automation, and AI-tool leads hidden inside saved reels.

## What It Does

- Stores the working library in browser `localStorage`.
- Imports normalized Instagram export JSON generated by the included parser.
- Supports manual link capture for reels, GitHub repositories, docs, and tools.
- Searches across URLs, captions, summaries, notes, tags, and extracted links.
- Highlights likely MCP, TradingView, GitHub, and high-priority automation leads.
- Exports JSON backups and Markdown/GitBook-style notes.
- Keeps raw Instagram exports, processed import output, local exports, `node_modules`, and build output out of Git.

## Project Layout

```text
src/                 React app source
src/lib/library.js   local storage, normalization, filtering, and export helpers
src/data/            sample seed items for first run/demo mode
scripts/             PowerShell helpers and Instagram export parser
supabase/            future cloud-sync schema, not required for the local MVP
data/raw/            ignored folder for Instagram export ZIPs or extracted folders
data/processed/      ignored folder for normalized parser output
exports/             ignored folder for exported notes/backups
index.html           Vite HTML entrypoint
vite.config.js       Vite dev/preview config
```

## Requirements

- Windows PowerShell for the helper scripts.
- Node.js 20+ and npm for dependency installation, or Codex Desktop with its bundled runtime plus an available npm CLI.
- Python 3.11+ for parsing Instagram exports, or Codex Desktop with its bundled Python runtime.

## Run Locally

From PowerShell:

```powershell
cd D:\AI\ai-reels-library
.\scripts\install-deps.ps1
.\scripts\start-local-server.ps1
```

Open:

```text
http://127.0.0.1:4173
```

For phone access on the same Wi-Fi, replace `127.0.0.1` with your computer LAN IP.

You can also use standard npm commands:

```powershell
npm install
npm run dev
```

## Import Instagram Data

Put an Instagram export ZIP or extracted export folder under:

```text
D:\AI\ai-reels-library\data\raw
```

Normalize it:

```powershell
cd D:\AI\ai-reels-library
.\scripts\parse-instagram-export.ps1 .\data\raw\instagram-export.zip
```

This creates:

```text
data\processed\reels-normalized.json
```

Then open the app and use **Import JSON** to merge the normalized items into local browser storage.

## Daily Workflow

1. Save or DM reels on Instagram.
2. Export Instagram data and place the ZIP in `data/raw/`.
3. Run `.\scripts\parse-instagram-export.ps1` against the ZIP or extracted folder.
4. Import `data\processed\reels-normalized.json` in the app.
5. Review MCP Leads, GitHub Repos, and Unprocessed views.
6. Add summaries, tutorial notes, tags, and status updates.
7. Export JSON for backups or Markdown for documentation.

## Build

```powershell
cd D:\AI\ai-reels-library
.\scripts\build-app.ps1
```

Or:

```powershell
npm run build
```

Build output is written to `dist/`, which is ignored by Git.

## Data And Privacy

The MVP is local-only:

- Imported items live in this browser's `localStorage`.
- Raw Instagram exports stay under `data/raw/`.
- Normalized parser output stays under `data/processed/`.
- Markdown or JSON files exported by the user can be kept under `exports/`.
- None of those local data folders are tracked by Git.

Use **Export JSON** in the app after important imports or reviews so you have a backup outside browser storage.

## Later Scale Path

When this grows beyond a local-only library, `supabase/schema.sql` can be reused as the starting point for cloud sync. Supabase setup is intentionally not required for the current MVP.
